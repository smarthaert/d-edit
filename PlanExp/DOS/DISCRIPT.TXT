                1. Содержимое инстолляционной дискеты

     Корневой каталог инстолляционной дискеты содержит следую-
     щие файлы:

     - DISCRIPT.TXT - данный файл;
     - README.TXT   - текстовый   файл  с  описанием процедуры
                      инстолляции  лабораторного  практикума и
                      требований к техническим средствам;
     - INSTALL.BAT  - командный  файл  для  установки  лабора-
                      торного практикума;
     - COP.EXE      - саморазворачивающийся архив,  содержащий
                      программное обеспечение практикума.
     - PPLEGA.EXE   - экранный драйвер-руссификатор.

     В каталоге  PLPR содержится подсистема формирования вари-
     антов заданий для лабораторных работ. Она содержит следу-
     ющие файлы:

     - README.TXT   - текстовый   файл  с  описанием процедуры
                      инстолляции подсистемы;
     - INSTALL.BAT  - командный файл для установки подсистемы;
     - PREP.EXE     - саморазворачивающийся архив,  содержащий
                      программное обеспечение подсистемы.


                   2. Назначение практикума

     Практикум предназначен для студентов ВУЗов,  для перепод-
готовки специалистов,  а также для  самостоятельного  изучения
методов планирования и обработки результатов сложных многофак-
торных экспериментов.  Ориентирован  на  поддержку  дисциплин,
изучающих  и  использующих  вероятностно-статистические методы
обработки данных,  методы подготовки,  проведения и  обработки
результатов  экспериментальных исследований.  Такие дисциплины
присутсвуют практически по всем  направлениям  технической,  а
также на ряде специальностей гуманитарной подготовки.
     Лабораторный практикум "Планирование экспериментов" вклю-
чает в себя цикл из пяти лабораторных работ:
     - полные факторные эксперименты;
     - дробные  факторные  эксперименты;
     - ортогональные центральные композиционные планы;
     - ротатабельные центральные композиционные планы;
     - униформные центральные композиционные планы.
     Каждая лабораторная работа представляет собой иммитацион-
ный эксперимент над некоторым объектом, характеристики которо-
го определяются данными конкретного варианта задания. На осно-
ве ограниченных  сведений  об  объекте  исследования,  которые
включают в  себя  количество факторов объекта,  его допустимую
область функционирования и некоторые априорные сведения о воз-
можной структуре модели, которую необходимо построить, обучае-
мый выбирает тип наиболее эффективного эксперимента,  осущест-
вляет выбор его области планирования,  строит план эксперимен-
та, проводит его и осуществляет полную статистическую обработ-
ку экспериментальных данных. Классическая схема статистической
обработки результатов регрессионного эксперимента предусматри-
вает выполнение следующих этапов:
     - вычисление точечных оценок числовых характеристик изме-
ренных значений отклика объекта исследования (вычисление сред-
них и оценок  дисперсий  отклика  во  всех  точках  факторного
пространства);
     - проверку статистической  гипотезы  о  воспроизводимости
эксперимента (на  этом  этапе обучаемый должен подобрать соот-
ветствующий проверяемой гипотезе статистический критерий  -  в
данном случае это критерий Кохрена, вычислить наблюдаемое зна-
чение этого критерия,  определить по  встроенным  в  практикум
статистическим таблицам  критическое  значение критерия и сде-
лать вывод о результате проверки гипотезы);
     - определение оценки дисперсии шума при проведении экспе-
римента;
     - вычисление  оценок  параметров строящейся регрессионной
модели, получаемые на основе использования  метода  наименьших
квадратов (МНК);
     - оценку дисперсий МНК-параметров модели;
     - проверку с помощью критерия Стьюдента значимости оценок
параметров модели (здесь тоже осуществляется  выбор  статисти-
ческого критерия  и определение его наблюдаемых и критического
значения);
     - коррекцию  модели,  осуществляемую  на  основе проверки
значимости;
     - проверку  с  помощью критерия Фишера адекватности полу-
ченной модели.
     Все перечисленные  действия  проделываются обучаемым,  за
исключением многократно повторяющихся однотипных операций, ко-
торые характерны для статистической обработки.  При выполнении
таких действий однократно проверяется умение обучаемого выпол-
нить его, а далее подобные операции выполняются автоматически.
     В результате  выполнения  лабораторных  работ  практикума
обучаемый должен научиться следующему:
     - правильно выбирать эффективные типы регрессионных  экс-
периментов по ограниченным данным об объекте исследования;
     - уметь провести выбор области экспериментирования и про-
водить нормализацию факторов объекта;
     - уметь составлять  планы  основных  типов  регрессионных
экспериментов для  получения  линейных  по  параметрам моделей
первого и второго порядков;
     - правильно  интерпретировать данные плана эксперимента и
проводить на его основе эксперимент над объектом исследования;
     - знать  классическую схему обработки результатов регрес-
сионных экспериментов и последовательность выполнения ее  эта-
пов;
     - уметь определять точечные оценки таких числовых  харак-
теристик случайных величин,  как оценки математического ожида-
ния и дисперсии,  иметь представдение о числе степеней свободы
дисперсионных оценок;
     - уметь пользоваться математическим аппаратом метода наи-
меньших квадратов при вычислении оценок параметров регрессион-
ных моделей при ортогональных и неортогональных планах  экспе-
риментов;
     - знать области применения статистических  критериев  при
проверке статистических гипотез различного вида (критерии Кох-
рена, Стьюдента, Фишера);
     - уметь  вычислять  наблюдаемые  значения и определять по
статистическим таблицам  критические  значения  статистических
критериев и интерпретировать их при принятии решения по прове-
ряемой гипотезе;
     - знать  и правильно использовать терминологию и основные
понятия и определения теории вероятностей, математической ста-
тистики и теории планирования экспериментов.
     В результате обучаемый должен уметь самостоятельно подго-
тавливать, планировать,  проводить  и  обрабатывать результаты
сложных многофакторных регрессионных экспериментов  различного
вида.
     Эффективность использования  компьютера  при  обучении  в
данной предметной  области  обусловлена  следующими причинами.
Во-первых,  подготовка и проведение работ на реальных объектах
или  на  их  физических моделях требует больших материальных и
трудовых затрат на создание специальных аппаратных  средств  и
на  приобретение  измерительной аппаратуры,  чего не требуется
при проведении компьютерного эксперимента над  математическими
моделями объектов.  Во-вторых,  использование компьютера и его
вычислительных возможностей избавляет обучаемого от  огромного
количества однотипных рутинных вычислительных операций,  кото-
рые являются характерным признаком любого метода  статистичес-
кой  обработки данных,  позволяя сосредоточить внимание на со-
держательной стороне проблемы,  что усиливает предметную моти-
вацию процесса обучения. В-третьих, использование при проведе-
нии работ контрольных заданий и вопросов со стороны компьютера
обеспечивает  детальный  контроль  степени  усвоения не только
практических навыков, но и уровня теоретических знаний обучае-
мых - все это практически без участия преподавателя.

                     3. Состав практикума

     В состав  программного  комплекса входит сам лабораторный
практикум и подсистема формирования вариантов  заданий.
     Программное обеспечение лабораторного практикума устанав-
ливается на  рабочие  места  обучаемых.  Запуск  лабораторного
практикума осуществляется с помощью командного файла PLAN.BAT.
     Подсистема формирования вариантов заданий устанавливается
на компьютере преподавателя.  В результате работы этой подсис-
темы создается файл  вариантов  заданий  VARIANT.DAT,  который
должен быть  переписан  на  рабочие  места обучаемых в каталог
WORK рабочего  каталога,  в  котором  установлен  лабораторный
практикум.

              4. Формирование вариантов заданий

     Каждое задание представляет собой следующий набор данных:
     - значения  параметров  идеальной  математической  модели
       объекта исследования,  над которым при проведении лабо-
       раторной работы будет проводиться иммитационный  экспе-
       римент;
     - допустимую область значений факторов объекта;
     - значение  среднего  квадратического  отклонения  ошибки
       эксперимента для настройки датчика нормально  распреде-
       ленных случайных  чисел,  возмущающих  значения отклика
       объекта при проведении его измерения;
     - тип плана эксперимента,  предназначенного для получения
       модели объекта в данном варианте задания.
     Модель идеального  объекта представляет собой линейную по
параметрам регрессионную модель  с  полиномиальными  базисными
функциями с количеством факторов от 2 до 5.  Такое ограничение
размерности объекта исследования обусловлена тем, что, с одной
стороны, обеспечивает не очень большую трудоемкость выполнения
лабораторных работ,  с другой стороны, позволяет продемонстри-
ровать и проверить все особенности влияния размерности объекта
исследования на  методику  построения  планов   экспериментов.
Дальнейшее увеличение  размерности объекта не несет в себе ка-
чественно новой информации.
     Параметры модели объекта в подсистеме обозначаются индек-
сами факторов,  входящих в базисную функцию данного параметра.
Например: параметр при факторе Х1 обозначается 1; параметр при
взаимодействии факторов Х2 и Х3 обозначается 23;  параметр при
квадрате фактора  Х3 обозначается 33;  постоянная составляющая
модели обозначается 0.
     При формировании вариантов заданий для проведения дробных
факторных экспериментов необходимо  придерживаться  следующего
правила: количество  отличных от 0 параметров идеальной модели
объекта должно быть точно  равно  числу  точек  спектра  плана
дробного эксперимента.
     Для искажения  идеальной  модели  первого  порядка  можно
ввести  в нее дополнительные параметры соответствующие квадра-
там факторов (в том числе и в дробном факторном эксперимента -
это  не  будет  противоречить сформулированному выше ограниче-
нию).
     Подсистема обеспечивает выполнение следующих функций:
     - просмотр готовых вариантов заданий (либо по их номерам,
       либо по шаблонам заданий);
     - формирование новых вариантов заданий;
     - коррекцию уже имеющихся заданий;
     - формирование новых вариантов на  основе  уже  имеющихся
       (для этого надо скопировать уже имеющийся вариант зада-
       ния, а затем провести коррекцию его данных и  сохранить
       созданный вариант);
     - сохранение созданных вариантов в файле вариантов  VARI-
       ANT.DAT (для  использования  новых  вариантов заданий в
       лабораторных работах надо переписать этот файл на рабо-
       чие места обучаемых);
     - вывод  перечня  вариантов  заданий  в укороченной форме
       в текстовый файл VARIANT.TXT (этот каталог используется
       преподавателем для  выдачи  номеров  вариантов  заданий
       обучаемым).

          5. Основные  характеристики практикума

     Практикум содержит полностью реализованную исполнительную
часть лабораторного практикума, обеспечивающую выполнение всех
лабораторных работ.
     Для упрощения проведения необходимых при выполнении работ
вычислений практикум оснащен встроенным резидентным  калькуля-
тором,  поддерживающим  выполнение практически всех инженерных
функций.  Загрузка калькулятора выполняется автоматически  при
запуске  командного  файла  лабораторного практикума PLAN.BAT.
Если выполнение лабораторной работы прерывается по  какой-либо
причине, то при повторном запуске практикума попытка повторной
установки калькулятора будет прервана. В ответ на соответству-
ющее  сообщение  необходимо  нажать любую клавишу клавиатуры и
продолжить запуск практикума. Калькулятор остается установлен-
ным после первой загрузки и им можно пользоваться, несмотря на
выданное сообщение.  Активизация  калькулятора  осуществляется
одновременным нажатием следующих клавиш:

                <Ctrl> + <Правый Shift> + <C>

     Калькулятор можно  перемещать по экрану освобождая закры-
ваемые им данные (перемещение осуществляется указателем  мани-
пулятора  "мышь" путем захвата верхней синей линейки калькуля-
тора, как в WINDOWS).
     При выполнении  лабораторных  работ  ведется  специальный
протокол,  в котором сохраняются все данные полученные при вы-
полнении  работы  или  введенные обучаемым.  В нем фиксируются
ошибки,  допущенные обучаемым,  а также обращения к  обучающей
подсистеме  с фиксацией их принадлежности к выполняемому этапу
работы, что позволяет получить детальную картину хода выполне-
ния работы и оценить степень подготовленности обучаемого.
     Кроме того протокол  позволяет  возобновить  прекращенный
сеанс  выполнения работы с точки прерывания и восстановить при
необходимости весь ход предыдущего сеанса. Для этого необходи-
мо после запуска практикума загрузить данные из сохраненного в
предыдущем сеансе файла-протокола.
     При аварийном  прерывании  работы по техническим причинам
или из-за необнаруженных пока ошибок в программном обеспечении
(при этом сохранение данных в файле-протоколе не было выполне-
но) следует повторно запустить практикум на выполнение  и  во-
зобновить  работу  с  помощью  позиции основного меню РАБОТА -
ПРОДОЛЖИТЬ, не используя позицию меню ЗАГРУЗИТЬ.
     Практикум содержит:
     - справочно-обучающую подсистему (позиция  меню  ПОМОЩЬ),
которая представляет из себя гипертекстовую систему,  содержа-
щую сведения теоретического и методического характера, а также
необходимые примеры выполнения различных операций. Контекстное
обращение к этой системе возможно из любой точки  лабораторной
работы;
     - подсистему вывода сводных данных по выполняемой  работе
и характеристик хода ее выполнения (ошибки,  обращения к спра-
вочной системе).
     Программное обеспечение  практикума  способно  работать в
среде WINDOWS и даже значительно более эффективно, чем в среде
MS DOS.
     Если лабораторный практикум не работает  с  установленным
на  Вашей  машине экранным драйвером-руссификатором,  то можно
перед запуском практикума загрузить имеющийся на дискете драй-
вер PPLEGA.EXE. Загрузка этого драйвера потребуется также, ес-
ли Вы захотите работать с практикумом в среде WINDOWS.

                    6. Контрольные примеры

     Практикум содержит 9 контрольных примеров,  записанных  в
следующих файлах протоколах (каталог \CONPRIM):
     - PFE.2   - полный факторный эксперимент - 2 фактора;
     - PFE.3   - полный факторный эксперимент - 3 фактора;
     - PFE.5   - полный факторный эксперимент - 5 факторов;
     - DFE.3   - дробный  факторный эксперимент - 3 фактора;
     - DFE.4   - дробный  факторный эксперимент - 4 фактора;
     - DFE.5   - дробный  факторный эксперимент - 5 факторов;
     - OCKP.2  - ортогональный центральный композиционный план
                 - 2 фактора;
     - RCKP.2  - ротатабельный центральный композиционный план
                 - 2 фактора;
     - UCKP.2  - униформный  центральный  композиционный  план
                 - 2 фактора.
     Для просмотра контрольных примеров необходимо  после  за-
пуска лабораторного практикума загрузить файл протокол необхо-
димого контрольного примера (позиция меню ФАЙЛ - ЗАГРУЗИТЬ), а
затем с  помощью позиций меню РАБОТА - ПОДГОТОВКА начать прос-
мотр этого контрольного примера.  При этом будут  представлены
все шаги  выполненной работы с уже введенными ответами обучае-
мого, за исключением самого процесса проведения  эксперимента.
При просмотре  контрольного примера можно делать попытки изме-
нить введенные данные.  При этом в ряде случаев (например, при
изменении числа параллельных опытов,  спектра плана или норми-
ровки факторов потребуется проведение  нового  эксперимента  и
обработки его  результатов в соответствии с внесенными измене-
ниями.
