           [1] Общий алгоритм работы
          ===========================

  1. Создать каталог исходных данных олимпиады
  2. Создать в этом каталоге файл TASKS.TXT содержащий :
      - Тексты задач
      - Пример теста и ответа на него к каждой задаче
         (Файлы INPUT.TXT и OUTPUT.TXT)
      - Время на выполнение каждого теста
     Примечание: Задачи именуются латинскими буквами "A","B",...
  3. Создать в каталоге подкаталог SOURCE с решениями задач
    на Pascal'е (.PAS), C++ (.CPP) или Basic'е (.BAS)
    И их откомпилированными вариантами.
  4. Создать каталог TESTS с подкаталогами A,B,...
  5. В каталогах TESTS\A,TESTS\B,...
    создать файлы тестов INPUT01.TXT,INPUT02.TXT...
   (вручную или при помощи программы TESTS.EXE )
    а также файл TESTS.TXT содержащий:
      - Количество тестов
      - Время на каждый тест
  6. Создать файлы ответов при помощи утилиты BUILD
  7. Создать файл USERS.TXT содержащий названия команд.
     (Каждая команда в своей строчке)
  8. Создать файл STARTUP.TXT в котором указать параметры олимпиады.
    UsersDir <Путь к каталогу пользователей>
  9. Запустить на одном из копьютеров SERVER.EXE
 10. Запустить на другом (других) компьютерах JUDGE.EXE

     [2] Взаимодействие пользователей с сервером
    =============================================

   [2.1] Посылка решений
     Пользователь копирует свое решение в
    "Личный каталог пользователя"\"Имя задачи".PAS (Borland Pascal 7.0) или
    "Имя задачи".CPP (Borland C++) или "Имя задачи".BAS (TurboBasic)
   [2.2] Посылка сообщений
     Пользователь копирует свое сообщение в
    "Личный каталог пользователя"\ADMIN.TXT
   [2.3] Приход сообщений
     Сообщения приходят в
    "Личный каталог пользователя"\MESSAGES.TXT
   [2.4] Текст задач хранится в INFO\TASKS.TXT
   [2.5] Отображение текущей таблицы результатов в INFO\MONITOR.TXT

          [3] Работа с судьей
         =====================

   [3.1] Регистрация судьи у сервера

  1) Судью следует запускать с параметром - путем к общему каталогу судей
  2) Судья записывает пустой файл REGISTER.TXT
  3) Сервер обнаруживает REGISTER.TXT и отвечает в тот же каталог файлом
    REGBACK.TXT содержащим :
      - Личный рабочий каталог судьи
      - Код занятости судьи
      - Код свободности судьи
  4) Судья читает REGBACK.TXT и уничтожает оба файла.

   [3.2] Завершение работы судьи

  1) Сервер создает пустой файл "Личный каталог"\UNREG.TXT
  2) Судья обнаруживает этот файл, уничтожает свой личный каталог и
    выходит в операционную систему.

   [3.3] Статус судьи

  Статус судьи обозначается строкой в файле "Личный каталог"\STATUS.TXT

   [3.4] Задание от сервера

  1) Задание от сервера приходит в файл "Личный каталог"\TASK.TXT :
      - Имя файла задачи
      - Имя компилятора
      - Каталог тестов
      - Количество тестов
      - Время на кажый тест (в секундах)
  2) Судья создает под собой каталог TEMP и копирует туда исходный
   текст программы как SOURCE.TXT
  3) Судья вызывает компилятор и получает SOURCE.EXE
  4) Судья копирует последовательно каждый из тестов как INPUT.TXT
  5) Судья запускает SOURCE.EXE
  6) Проверяет ответ программы
  7) Уничтожает каталог TEMP
  7) Выдает файл результата RESULTS.TXT
      - Количество пройденных тестов
      - Номер ошибки
       1 - Все благополучно
       2 - Ошибка компиляции
       3 - Ошибка выполнения
       4 - Неправильный ответ

          [4] Работа с консолью
         =======================

 Help - Вызов помощи на экран
 UsersDir,JudgesDir,WorkDir,TasksDir,InfoDir <Каталог> - Задать каталог
 Info - Вывод информации о текущих установках системы
 AddUser <Имя Пользователя> - Добавление пользователя
 DelUser <Номер Пользователя> - Удаление пользователя
 DelAllUsers - Удаление всех пользователей
 UsersList - Список пользователей
 OpenUsers,CloseUsers - Открыть/Закрыть каталог пользователей
 Start <Время соревнования в минутах> - Запуск соревнования
 LastProblem <Имя последней задачи>
 JudgesList - Список судей
 Execute <Имя файла> - Исполнение командного файла
 TasksList - Список запущенных обьектов
 ShutDown - Выход из системы
